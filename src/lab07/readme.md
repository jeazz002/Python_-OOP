# Лабораторная работа №7: Консольное приложение (CLI)

## 1. Цель работы

- проектирование архитектуры приложения с разделением на слои;
- реализация интерактивного меню с обработкой пользовательского ввода;
- работа с файлами (сохранение/загрузка в JSON);
- использование собственных исключений;
- применение аннотаций типов и generic-коллекций.

## 2. Структура проекта

### Назначение файлов
- **`exceptions.py`** – определяет собственные исключения для чёткой обработки ошибок предметной области.
- **`models.py`** – описывает сущность «Пациент» (имя, возраст, диагноз, статус лечения).
- **`collection.py`** – обобщённая коллекция с методами `add`, `remove`, `find`, `filter`, `map`, `sort_by`.
- **`storage.py`** – отвечает за сохранение коллекции в JSON и загрузку из файла.
- **`app.py`** – слой бизнес-логики: добавление, удаление, поиск, фильтрация, сортировка; не содержит ввода/вывода.
- **`cli.py`** – пользовательский интерфейс: вывод меню, чтение команд, валидация ввода, вывод таблицы пациентов, вызов методов `app`.
- **`main.py`** – запускает CLI.

## 3. Описание CLI

После запуска (`python main.py`) пользователю предлагается меню:

- **Добавление пациента**: запрашивает имя, возраст, диагноз, статус лечения (вылечен/не вылечен). Проверяется уникальность имени, корректность возраста (0–150), непустые поля.
- **Вывод всех пациентов**: отображает таблицу с колонками: Имя, Возраст, Диагноз, Статус.
- **Поиск по имени**: точное совпадение имени, выводит информацию о найденном пациенте.
- **Фильтрация по диагнозу**: показывает всех пациентов с указанным диагнозом.
- **Сортировка**: выбор критерия (имя/возраст/диагноз) и направления (по возрастанию/убыванию). После сортировки список обновляется.
- **Удаление по имени**: запрашивает подтверждение (y/n), после удаления коллекция изменяется.
- **Сохранение и выход**: все изменения записываются в `patients.json`, программа завершается.

### Обработка ошибок
- При вводе номера пункта меню, отличного от 1–7, выводится сообщение «Неверный пункт меню».
- При вводе букв вместо числа в запросе возраста используется `get_int_input()`, которая повторяет запрос до получения целого числа.
- Пустые имена и диагнозы отклоняются с сообщением.
- Попытка добавить пациента с уже существующим именем вызывает `DuplicateItemError`, который перехватывается и выводится понятное сообщение.
- Удаление несуществующего пациента – `ItemNotFoundError`.
- Некорректный ключ сортировки – `ValueError`.

### Сохранение и загрузка
- При запуске приложение автоматически загружает данные из `patients.json`.
- При выборе пункта «Сохранить и выйти» текущее состояние коллекции сохраняется в тот же файл.
- Формат – JSON, структура: список словарей с ключами `name`, `age`, `diagnosis`, `is_treated`.
